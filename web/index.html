📚 Distributed Systems & CAP Theorem Playground

Consistency vs Availability (AP vs CP)

Writes (CP system): To prevent conflicting codes, writes are routed strictly to the primary DB in US-East. If US-East app server or database goes offline, writes fail. We prioritize Consistency over Availability for writes.

Reads (AP system): Reads are served by regional replicas/caches. If the database replica or cache fails, the server falls back to cross-region reads. If a region goes completely dark, the router routes to the next closest region. We prioritize Availability for reads.

Eventual Consistency (Replica Lag)

Replication from the Primary DB (US-East) to Read Replicas is asynchronous to avoid stalling write requests. In Mock Mode, we simulate a replication delay (1.5s for US-West, 3.0s for EU-West).

Try this: Write a new URL and immediately request it from EU-West. You'll trigger a Primary DB Fallback or see eventual consistency in action as the replica catches up!
